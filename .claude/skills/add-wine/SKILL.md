# Add a New Wine Biome

Wine is biome-driven in [wine.py](../../../wine.py). There are no named wine "types" — each biome grows a distinct grape variety with a unique base profile. Adding a new biome means adding it to `BIOME_GRAPE_PROFILES`, `BIOME_DISPLAY_NAMES`, and `VARIETY_DISPLAY_NAMES`.

Individual grapes are `Grape` dataclass instances generated by `WineGenerator`. The generator reads `BIOME_GRAPE_PROFILES` directly — no generator changes needed.

## Step 1 — Add a profile to BIOME_GRAPE_PROFILES

Add the new biome key to `BIOME_GRAPE_PROFILES` in [wine.py](../../../wine.py). Each attribute is a base value 0.0–1.0; the generator applies a small gaussian jitter per grape.

```python
BIOME_GRAPE_PROFILES = {
    ...
    "coral_reef": {
        "sweetness":  0.75,   # ripe, tropical fruit
        "acidity":    0.90,   # very bright
        "tannin":     0.10,   # almost none — delicate
        "body":       0.35,   # light
        "aromatics":  0.90,   # intensely floral
        "variety":    "viognier",
    },
}
```

**variety** is a string key for the grape cultivar — lowercase with underscores. Choose an existing one or coin a new one.

**Profile guidance by style:**
- High tannin + body → red-leaning (cabernet, syrah style)
- High acidity + aromatics, low tannin → white-leaning (riesling, sauvignon blanc style)
- High sweetness + low stop-time in fermentation → dessert potential

## Step 2 — _CODEX_BIOMES (no change needed)

`_CODEX_BIOMES` is already derived from `BIOME_GRAPE_PROFILES.keys()`, so `WINE_TYPE_ORDER` updates automatically when you add the profile in Step 1.

## Step 3 — Add to BIOME_DISPLAY_NAMES

```python
BIOME_DISPLAY_NAMES = {
    ...
    "coral_reef": "Coral Reef",
}
```

## Step 4 — Add to VARIETY_DISPLAY_NAMES

```python
VARIETY_DISPLAY_NAMES = {
    ...
    "viognier": "Viognier",
}
```

## Style and flavor mechanics (no changes needed)

Style (red/white/rosé/sparkling/dessert) is determined at fermentation by `determine_style()` based on crush style, yeast, and stop timing. The biome profile influences which style is *likely* but the player's choices decide it.

Flavor notes appear automatically based on which attributes exceed 0.6 after processing:

| Attribute | Notes drawn from (if > 0.6) |
|-----------|-----------------------------|
| `sweetness` | ripe cherry, honey, fig, candied plum, vanilla, brown sugar |
| `acidity` | citrus zest, green apple, white peach, lime pith, cranberry, verjuice |
| `tannin` | leather, graphite, dark chocolate, cedar shavings, walnut skin, dried tea |
| `body` | blackberry jam, cocoa, espresso, plum compote, stewed fruit, smoke |
| `aromatics` | violet, rose petal, elderflower, jasmine, lavender, orange blossom |

## Verification checklist

- [ ] Biome key added to `BIOME_GRAPE_PROFILES`
- [ ] Same key added to `BIOME_DISPLAY_NAMES`
- [ ] Variety key added to `VARIETY_DISPLAY_NAMES`
- [ ] Biome key matches the key used in [biomes.py](../../../biomes.py)
- [ ] Harvest grapes in the new biome — grape should generate and be processable through the full winery pipeline
