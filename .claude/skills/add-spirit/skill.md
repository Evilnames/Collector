# Add a New Spirit Biome

Spirits are biome-driven in [spirits.py](../../../spirits.py). Each biome grows a distinct grain type that produces one spirit type (whiskey, bourbon, rum, gin, brandy, or vodka). Adding a new biome means adding it to `BIOME_SPIRIT_PROFILES` and `BIOME_DISPLAY_NAMES`.

Individual spirits are `Spirit` dataclass instances generated by `SpiritGenerator`. The generator reads `BIOME_SPIRIT_PROFILES` directly — no generator changes needed.

## Step 1 — Add a profile to BIOME_SPIRIT_PROFILES

Add the new biome key to `BIOME_SPIRIT_PROFILES` in [spirits.py](../../../spirits.py). Each attribute is a base value 0.0–1.0; the generator applies a small gaussian jitter per spirit.

```python
BIOME_SPIRIT_PROFILES = {
    ...
    "coral_reef": {
        "grain_type":      "sugarcane",   # see grain types below
        "spirit_type":     "rum",         # see spirit types below
        "grain_character": 0.20,          # malt / grain intensity
        "sweetness":       0.90,          # caramel, vanilla, tropical
        "spice":           0.30,          # pepper, clove, cinnamon
        "smokiness":       0.05,          # peat, char, campfire
        "smoothness":      0.80,          # silk, cream, rounded malt
    },
}
```

**grain_type** — sets the source material label shown in the collection detail panel:
- `"grain"` — barley/wheat, suits whiskey or vodka
- `"corn"` — corn mash, suits bourbon
- `"sugarcane"` — tropical cane, suits rum
- `"botanical"` — juniper and wildflower infusion, suits gin
- `"pomace"` — grape pomace (wine byproduct), suits brandy

**spirit_type** — determines which output items are generated and which buff is applied on drinking:

| spirit_type | Buff | Effect |
|-------------|------|--------|
| `"whiskey"` | grit | Pick power +1.5 / Mining speed +20% |
| `"bourbon"` | warmth | Cold damage -60% / Hunger drain -20% |
| `"rum"` | sea_legs | Move speed +20% / Jump height +1 |
| `"gin"` | clarity | Collect radius +60% / Discovery XP +15% |
| `"brandy"` | refinement | Discovery XP +30% / Craft speed +20% |
| `"vodka"` | endurance | Hunger drain -50% / Stamina +25% |

**Profile guidance by spirit type:**

| Spirit | grain_character | sweetness | spice | smokiness | smoothness |
|--------|----------------|-----------|-------|-----------|-----------|
| Whiskey | high (0.65–0.85) | low–mid | high (0.75+) | mid (0.30–0.50) | low–mid |
| Bourbon | high (0.70–0.85) | mid–high | mid–high | low–mid | low–mid |
| Rum | low (0.20–0.35) | high (0.75+) | low–mid | very low | mid–high |
| Gin | very low (0.20–0.30) | low–mid | mid–high | very low | mid–high |
| Brandy | low–mid (0.35–0.45) | mid–high | low–mid | low | mid–high |
| Vodka | low–mid | very low | very low | very low | very high (0.85+) |

## Step 2 — _CODEX_BIOMES (no change needed)

`_CODEX_BIOMES` is already derived from `BIOME_SPIRIT_PROFILES.keys()`, and `SPIRIT_TYPE_ORDER` is built from it — both update automatically when you add the profile in Step 1.

## Step 3 — Add to BIOME_DISPLAY_NAMES

```python
BIOME_DISPLAY_NAMES = {
    ...
    "coral_reef": "Coral Reef",
}
```

## Flavor mechanics (no changes needed)

Flavor notes appear automatically based on which attributes exceed 0.6 after distillation and aging:

| Attribute | Notes drawn from (if > 0.6) |
|-----------|-----------------------------|
| `grain_character` | malt, cereal, bread crust, biscuit, dried grain, toasted barley |
| `sweetness` | vanilla, caramel, honey, ripe banana, brown sugar, molasses |
| `spice` | black pepper, clove, nutmeg, cinnamon, dried chili, white pepper |
| `smokiness` | peat, campfire, char, ash, smoked oak, bonfire |
| `smoothness` | cream, silk, gentle oak, soft finish, velvety, rounded malt |

Barrel notes are added on top when the player ages the spirit:
- **new_oak** → fresh oak, tannin, sawdust, bold wood
- **charred_oak** → vanilla char, toasted coconut, caramel layer, sweet smoke
- **used_oak** → gentle oak, dried fruit, polished wood, soft tannin

## What happens automatically

Once the profile is added, the full pipeline works without further changes:

- `SpiritGenerator.generate(biome)` reads the profile and jitters attributes
- Harvesting `GRAIN_CROP_MATURE` in that biome creates a `Spirit` with the correct `spirit_type`
- The Copper Still, Barrel Room, and Bottling Station all work unchanged
- Output items (`whiskey`, `whiskey_aged`, `whiskey_reserve`, etc.) already exist in [items.py](../../../items.py)
- The Spirits Codex grid shows the new biome row with three tier columns (young / aged / reserve)
- `discovered_spirit_types` tracks `"{biome}_young"`, `"{biome}_aged"`, `"{biome}_reserve"` keys

## Verification checklist

- [ ] Biome key added to `BIOME_SPIRIT_PROFILES` with valid `grain_type` and `spirit_type`
- [ ] Same key added to `BIOME_DISPLAY_NAMES`
- [ ] Biome key matches the key used in [biomes.py](../../../biomes.py) (or is a new custom biome)
- [ ] Harvest grain crops in the new biome — spirit should generate with the correct type
- [ ] Spirit processable through Copper Still → Barrel Room → Bottling Station
- [ ] New biome row appears in the Spirits Codex (tab 9 in Encyclopedia)
- [ ] Bottled spirit grants the correct buff on consumption
